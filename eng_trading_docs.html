System Overview

This trading system is an algorithmic cryptocurrency trading platform designed to analyze market data in real-time, identify trading opportunities based on statistical indicators, and execute trades automatically. The system connects to cryptocurrency exchanges via WebSocket, processes market data, calculates various technical indicators, generates trading signals, and manages trades with risk management rules.

Key Features

  • Real-time market data processing via WebSocket connections
  • Statistical analysis of market conditions
  • Automated trading signal generation
  • Risk management with adaptive position sizing
  • Performance tracking and trade journaling
  • Event-driven architecture for flexibility and extensibility

System Architecture

The trading system follows a modular, event-driven architecture organized into several key modules:

Connectivity

Handles communication with exchanges via WebSocket connections, processes incoming market data, and maintains connection health.

Market

Stores and manages market data, calculates technical indicators and market metrics used for analysis.

Analysis

Analyzes market data, generates trading signals, and identifies entry/exit opportunities based on market conditions.

Trading

Manages active trades, handles position sizing, implements risk management, and tracks performance.

Events

Provides an event system for communication between components, enabling loose coupling and flexibility.

Utils

Contains utility functions, configuration management, and logging functionality.

Storage

Handles data persistence, recording market data and trade information for later analysis.

Reporting

Provides status updates and performance reports during system operation.

Data Flow & Function Calls

System Initialization Flow

  1. main.py - Entry point that initializes the system
  2. Logger setup via LoggerSetup.get_logger()
  3. Configuration loading via TradingConfig.load_from_file()
  4. MarketAnalyzer initialization with configuration parameters
  5. WebSocket manager initialization with the analyzer and symbols
  6. Event handlers registration for trade events
  7. Status reporter initialization and start
  8. WebSocket connection start based on command line arguments

Market Data Processing Flow

  1. WebSocket receives market data via _handle_message()
  2. Data is normalized and passed to analyzer.process_tick()
  3. MarketAnalyzer validates the data and emits a TICK event
  4. Market data is updated via market_data.add_tick()
  5. Market metrics are calculated via metrics_calculator.calculate_metrics()
  6. If metrics are updated, a METRIC_UPDATE event is emitted
  7. If warmup is complete, signal generation is triggered
  8. SignalGenerator analyzes conditions and may generate a signal
  9. If a signal is generated, it's processed by _process_signal()
  10. A SIGNAL event is emitted with the signal details

Trade Execution Flow

  1. When a BUY signal is processed, trade_manager.open_trade() is called
  2. Position size is calculated based on risk parameters
  3. Stop loss and take profit levels are calculated
  4. ActiveTrade is updated with trade details
  5. A TRADE_OPENED event is emitted
  6. When a CLOSE signal is processed, trade_manager.close_trade() is called
  7. Profit/loss is calculated
  8. Trade details are recorded in the trade journal
  9. Performance metrics are updated
  10. A TRADE_CLOSED event is emitted
  11. ActiveTrade is reset

Market Metrics Explained

The system uses several market metrics to analyze market conditions and make trading decisions:

Realized Volatility

Description: Measures the average price change percentage over time.

Calculation: Standard deviation of returns multiplied by the square root of (252 * 1440) to annualize, expressed as a percentage.

Trading Significance: Higher volatility may indicate potential trading opportunities but also increased risk.

Average True Range (ATR)

Description: Measures market volatility by decomposing the entire range of an asset price for a period.

Calculation: Average of true ranges over a 14-period window, where true range is the greatest of: current high - current low, current high - previous close, or current low - previous close.

Trading Significance: Used to set stop loss levels and position sizes based on market volatility.

Relative Strength

Description: Measures the strength of price movement, similar to RSI but normalized between 0 and 1.

Calculation: Ratio of gains to total absolute returns over a window period.

Trading Significance: Values above 0.5 indicate bullish momentum, while values below 0.5 indicate bearish momentum.

Order Imbalance

Description: Measures the ratio between bid and ask volumes.

Calculation: Bid volume divided by total volume (bid + ask).

Trading Significance: Values above 0.5 indicate more buying pressure, while values below 0.5 indicate more selling pressure.

Trend Strength

Description: Measures the strength and direction of the current price trend.

Calculation: Linear regression slope multiplied by R-squared value and scaled by price level.

Trading Significance: Positive values indicate uptrend, negative values indicate downtrend, and the magnitude indicates strength.

Market Efficiency Ratio (MER)

Description: Measures how efficiently price moves from one point to another.

Calculation: Net directional movement divided by the total path length over a 30-period window.

Trading Significance: Higher values indicate a more efficient, trending market, while lower values indicate a choppy, range-bound market.

Trading Logic

Buy Conditions

The system enters a buy trade when all of the following conditions are met:

  • Realized volatility is above the threshold (0.3%)
  • Relative strength is above the threshold (0.2)
  • Trend strength is positive and above the threshold (6)
  • Order imbalance is above the threshold (0.25)
  • Market efficiency ratio is above the threshold (1.0)

Exit Conditions

The system exits a trade when any of the following conditions are met:

  • Price hits the stop loss level
  • Price hits the take profit level
  • Trailing stop is triggered after profit exceeds activation threshold
  • Trade duration exceeds 4 hours
  • Trend strength reverses below -0.4

Risk Management

The system implements several risk management techniques:

  • Position Sizing: Default risk is 2% per trade, but can be adjusted based on market conditions
  • Stop Loss: Set at 1.5 times the ATR below entry price
  • Take Profit: Set at 2.5 times the stop distance (risk-reward ratio of 2.5)
  • Trailing Stop: Activated when profit reaches 1%, with distance of 1.5 times ATR
  • Adaptive Sizing: Adjusts position size based on volatility, trend alignment, and recent performance

How the System Buys

  1. System analyzes market metrics to identify favorable conditions
  2. When all buy conditions are met, a BUY signal is generated
  3. Position size is calculated based on risk parameters and market conditions
  4. Stop loss is set at 1.5 * ATR below entry price
  5. Take profit is set at entry price + (stop distance * 2.5)
  6. Trade is opened and TRADE_OPENED event is emitted
  7. Trade details are logged and tracked

How the System Sells

  1. For active trades, the system continuously checks exit conditions
  2. If price hits stop loss, take profit, or trailing stop, a CLOSE signal is generated
  3. If trend reverses or time limit is reached, a CLOSE signal is generated
  4. Trade is closed and profit/loss is calculated
  5. Trade details are recorded in the journal
  6. Performance metrics are updated
  7. TRADE_CLOSED event is emitted
  8. System returns to looking for new entry opportunities

Running the System

Prerequisites

  • Python 3.7 or higher
  • Required packages: websocket-client, numpy, scipy
  • Internet connection for live trading

Installation

Install the required dependencies:

pip install -r requirements.txt

Running in Live Mode

To run the system in live trading mode:

python run.py live

Or alternatively:

python -m TRADE.main live

Running in Backtest Mode

Backtest mode is not yet fully implemented, but the command would be:

python run.py backtest

System Output

When running, the system will display:

  • Connection status messages
  • Trade signals with market metrics
  • Trade opened/closed notifications with details
  • Error messages if any issues occur
  • Regular status updates from the status reporter

Stopping the System

To stop the system, press Ctrl+C in the terminal. The system will perform a graceful shutdown, closing any open connections and trades.

Configuration Options

The system can be configured by editing the trading_config.json file or by modifying the default values in TRADE/utils/config.py.

The buy and exit conditions can be configured in the MARKET_CONDITIONS section:

Buy Conditions

Parameter Description Default Value
volatility_threshold Minimum volatility for buy entry 0.3
relative_strength_threshold Minimum relative strength for buy entry 0.2
trend_strength Minimum trend strength for buy entry 6
order_imbalance Minimum order imbalance for buy entry 0.25
market_efficiency_ratio Minimum market efficiency for buy entry 1.0

Exit Conditions

Parameter Description Default Value
profit_target_multiplier Take profit distance as multiple of stop distance 2.5
trailing_stop_act_ivation Profit percentage to activate trailing stop 1.0
trailing_stop_distance Trailing stop distance as multiple of ATR 1.5

Important Note

Changing configuration parameters can significantly impact trading performance. It's recommended to test any changes in a simulated environment before using them in live trading.
